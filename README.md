Бодит цагийн шифрлэгдсэн болон шифрлэгдээгүй ярианаас текст рүү дамжуулах урсгал дахь хоцролтыг харьцуулах нь
# Монгол хэлний STT + Шифрлэлтийн Гүйцэтгэлийн Benchmark

> **Speech-to-Text (STT) системд зориулсан шифрлэлтийн алгоритмуудын гүйцэтгэлийн харьцуулалт**
> AES-128 · AES-256 · ChaCha20 · RSA-2048

---

##  Судалгааны зорилго

Дуу хоолойг бичвэрт буулгах (STT) системүүдэд өгөгдлийн нууцлалын шаардлага өсөн нэмэгдэж байгаа тул шифрлэлтийн гүйцэтгэлийн зардлыг нарийвчлан хэмжих нь практик ач холбогдолтой болоод байна.

Энэхүү судалгаанд:
- **Монгол хэлний** аудио файлуудыг Google STT-ээр текст болгон хөрвүүлж
- Хөрвүүлсэн текст дээр **4 шифрлэлтийн алгоритм**-ын гүйцэтгэлийг хэмжиж
- **6 өөр хэмжилтийн арга**-ар боловсруулалтын саатал (latency)-д үзүүлэх нөлөөг харьцуулан судалсан

##  Шифрлэлтийн алгоритмууд

| Алгоритм | Төрөл | Түлхүүрийн урт | Горим |
|----------|-------|----------------|-------|
| AES-128 | Симметр | 128 бит | CBC |
| AES-256 | Симметр | 256 бит | CBC |
| ChaCha20 | Симметр (урсгал) | 256 бит | — |
| RSA-2048 | Асимметр | 2048 бит | OAEP |

##  Хэмжилтийн аргууд

| # | Арга | Тайлбар |
|---|------|---------|
| 1 | **Олон давталт + статистик** | `time.perf_counter` ашиглан mean, median, std, min, max тооцоолно |
| 2 | **timeit** | GC-г унтраасан нөхцөлд цэвэр хурд хэмжинэ |
| 3 | **cProfile** | Функц бүрийн дуудалт тоо, дотоод цагийн задаргааг харуулна |
| 4 | **Memory Profiler** | Санах ойн дээд хэрэглээг MiB-ээр хэмжинэ |
| 5 | **Хэмжээ vs Саатал** | 64B–бодит хэмжээ хүртэл өгөгдлийн хэмжээний нөлөөг харуулна |
| 6 | **Matplotlib график** | Box plot, Bar chart, Line chart нэгтгэсэн харагдац |

```
crypto_benchmark.py
│
├── split_and_transcribe()     # Chunk-based STT (55с хэсгүүдэд)

│
├── run_aes128()               # Шифрлэлтийн алгоритмууд
├── run_aes256()
├── run_chacha20()
├── run_rsa2048()

├── method1_statistics()       # Олон давталт + статистик
├── method2_timeit()           # timeit хэмжилт
├── method3_cprofile()         # cProfile задаргаа
├── method4_memory()           # Санах ойн хэрэглээ

├── method6_plot()             # График үүсгэх

├── benchmark_one_file()       # Нэг файлын бүрэн benchmark
├── plot_combined_summary()    # Бүх файлын нэгтгэсэн график

```

## STT Chunk-based шийдэл

Google Speech Recognition API нь нэг хүсэлтэд **~60 секундын хязгаартай**. Урт аудио файл шууд илгээхэд алдаа эсвэл хэсэгчилсэн үр дүн гардаг тул файлыг **55 секундын хэсгүүдэд** хуваан тус бүрийг тусдаа хүсэлт болгон явуулна.

```
Record_9min.mp3  (540 секунд)
      │
      ├── chunk_0.wav  [0с – 55с]   → Google STT → текст₁
      ├── chunk_1.wav  [55с – 110с] → Google STT → текст₂
      ├── ...
      └── chunk_9.wav  [495с – 540с]→ Google STT → текст₁₀
                                              ↓
                               нэгтгэсэн транскрипт
```
